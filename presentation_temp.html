1. The Target: AES-128

Focus on the very first round, specifically the SubBytes (S-Box) operation.

Intermediate value \( v \) is calculated as:

\[ v = SBOX [ p \oplus k ] \]

AES Vulnerability

  • \( p \): Known plaintext byte.
  • \( k \): Unknown target key byte (0 to 255).
  • SBOX: Highly non-linear substitution table.
Why target the S-Box?
The strong non-linearity means incorrect key guesses will produce drastically different and unpredictable binary outputs, making the correct key stand out.

2. Leakage Models

Microcontrollers leak power proportional to the data they process in registers and buses.

Hamming Weight (HW)

The number of 1-bits transitioning in a register:

\[ HW(v) = \sum_{j=0}^{7} v_j \]

Example: \( HW(01100010_2) = 3 \)

3. Correlation Power Analysis

Compares real power traces (\( t_i \)) against a theoretical leakage model (\( h_i \)).

\[ h_i = HW(SBOX[p_i \oplus \hat{k}]) \]

where \( \hat{k} \) is our key guess.

Pearson Correlation

\[ \rho = \frac{\sum_{i=1}^N (t_i - \bar{t})(h_i - \bar{h})}{\sqrt{\sum_{i=1}^N (t_i - \bar{t})^2 \sum_{i=1}^N (h_i - \bar{h})^2}} \]

Correct Key

\( |\rho| \approx 1 \)

Incorrect Guess

\( \rho \approx 0 \) (Noise)

4. Differential Power Analysis

Instead of total bit flips, we target a single selection bit \( b \).

\[ D(p_i, \hat{k}, b) = \text{bit}_b(SBOX[p_i \oplus \hat{k}]) \in \{0, 1\} \]

Difference of Means (DoM)

Partition the measured traces horizontally based on \( D \):

Set 1: \( S_1 = \{ t_i \mid D = 1 \} \)
Set 0: \( S_0 = \{ t_i \mid D = 0 \} \)
\[ \Delta = \frac{1}{|S_1|}\sum_{i \in S_1} t_i - \frac{1}{|S_0|}\sum_{i \in S_0} t_i \]

The correct key yields the largest magnitude \( \Delta \) peak.

5. Hardware Acquisition

The Problem: Standard Arduino analogRead() is too slow and jittery for precise temporal alignment. 

// Free-Running ADC Setup (AVR)
// Aligns directly with AES execution timing

void setupADC() {
    ADMUX = (1 << REFS0) | (1 << ADLAR); 
    
    // Prescaler 16, Enable Interrupts, Auto Trigger
    ADCSRA = (1 << ADEN) | (1 << ADATE) | (1 << ADIE) | (1 << ADPS2);
    
    ADCSRB = 0; // Free running mode
}

6. Python Vectorization

Executing DPA rapidly over thousands of traces using NumPy: 


# DPA Partitioning in Py
selection = ((SBOX[plaintexts ^ key_guess] >> bit) & 1).astype(bool)

group1_mean = traces[selection].mean(axis=0)
group0_mean = traces[~selection].mean(axis=0)
difference_of_means = group1_mean - group0_mean

7. Key Takeaways

CPA (Correlation): Uses a complete leakage model (HW). Needs fewer traces.
DPA (Differential): Assumes minimal correlation (1 bit). Needs more traces.
Crucial Requirement: Signal-to-Noise Ratio (SNR) and strict temporal alignment are absolute requirements for both!
